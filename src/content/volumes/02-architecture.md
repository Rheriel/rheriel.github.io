This volume focuses on the words engineers use when they explain structure: where code belongs, how responsibilities are separated, and how trade-offs shape system design.

The first pass will cover SOLID, separation of concerns, dependency injection, composition, common design patterns, and architecture styles such as layered, clean, hexagonal, onion, and domain-driven design.

- SOLID
- Separation of Concerns
- Dependency Injection
- Composition vs Inheritance
- Repository Pattern
- Unit of Work
- Factory Pattern
- Strategy Pattern
- Adapter
- Decorator
- Observer
- Builder
- Layered Architecture
- Vertical Slice Architecture
- Clean Architecture
- Hexagonal Architecture
- Onion Architecture
- Domain-Driven Design
